socket.io-분산 서버 환경에서 사용하기

socket.io 서버가 대규모이거나, 부하가 심해 여러대로 늘려야하는 상황일 때 서버간 데이터를 공유해야 합니다.
socket.io 기본 store는 메모리 기반이여서 프로세스 하나에 데이터가 종속되어있기 때문에 redis를 이용해 분산 서버 환경에서 socket.io 데이터를 공유해야 합니다.

해결방법

socket.io를 redis에 연결해 publish / subscribe 이용해서 메세지를 전송, 수신 하는 방식으로 해결합니다. 공식 문서
해당 기능을 직접 구현하는 방법이 있지만 이미 완성된 모듈을 사용합니다. 두개 모듈을 지원하는데 이번 글은 첫번째 모듈 socket.io-redis를 이용합니다.

  • socket.io-redis
  • socket.io-adapter

기존 socket.io에서 자주 사용하는 대부분의 기능을 사용 할 수 있습니다. socket.io redis github 참고

사용 기술

  • socket.io
  • redis (ioRedis)
  • Node.js

해당 모듈을 이용해 연결, 사용하는 간단한 예제 입니다.

socket.io-redis 연결하기

  • redis, socket.io, socket.io-redis(adapter)를 사용하기 위해 선언해 줍니다.
  • 예제엔 빠져있지만 redis db 등 별개 설정이나, redis function override 등 기존 Redis와 똑같이 가능합니다.

Namespace 사용하기

  • 기존 socket.io 처럼 namespace 사용 가능합니다.

Client room 입장시키기

  • namepsace를 사용한 예제입니다.
  • remoteJoin 함수는 socket.io-redis github 참고

Room 접속한 client에게 메세지 전송하기

  • broadcast를 사용해서 전송하는 이유는 기존 io.to('roomname').emit... 를 사용해 메세지를 전송하면 특정 유저에게만 메세지가 전송되는 현상이 있습니다.
