루프백 리모트 메소드

리모트 메소드는 루프백의 표준 모델 REST API 제공하지 않는 REST API를 생성할 수 있습니다.

생성 방법

두가지 방법의 생성 방법을 다룹니다.

  • callback
  • async

callback 방식의 리모트 메소드 정의방법

리모트 메소드를 생성하는 생성 함수와 리모트 메소드를 처리하는 함수 두가지를 선언해야 합니다.

Model.remoteMethod 으로 리모트 메소드 REST API를 생성합니다.

accepts 사용자에게 입력받을 값들을 정의합니다.아래처럼 하나씩 파라미터를 지정해 줄 수 있습니다.
다른 모델을 파라미터로 받으려면 type에 다른 모델명을 작성해주시면 됩니다.

http REST API 생성할때 사용하는 설정값 입니다. verb는 REST API 규격에 맞는 method 형식 입니다.
path는 사용자에게 보여줄 엔드포인트 입니다. 예제에는 /으로 지정되어 있기 때문에 모델명/ 으로 나타냅니다.
path에 아무런 빈 값을 넣으면 리모트 메소드 선언 이름과 같이 선언됩니다. greet

returns REST API output을 정의하는 부분입니다. accepts와 같이 type를 모델로 지정가능합니다.

Model.greet 으로 리모트 메소드를 처리하는 함수를 작성합니다.

callback 으로 선언하면 파라미터로 msg, callback 두가지가 넘어옵니다.
msg에는 사용자가 요청을 보낼때 담은 데이터가 넘어오고 callback으로 리턴합니다.
callback함수를 실행할때 첫번째 파라미터는 에러를 담아주고 에러가 없으면 null을 넣어줍니다.
두번째 파라미터는 사용자에게 내보낼 output 파라미터 입니다.

Model.greet 으로 리모트 메소드를 처리하는 함수를 작성합니다. async

async형태로 선언하면 callback 함수 없이도 리모트 메소드 처리 함수를 선언할 수 있습니다.
